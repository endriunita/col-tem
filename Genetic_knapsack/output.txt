Knapsack problem using genetic algorithm:
I've implemented this after the Generational GA algorithm present in course 5. The algorithm has 3 parameters, if we're not taking the input file into consideration: population size,
number of generations and the mutation probability.The starting population is generated directly valid. The selection of parents consists in taking two random parents from the "old"
generation and creating two offsprings with random index, single-point crossover, after which I applied weak mutation that uses the mutation probability parameter.
Because my starting population was generated valid, I'm not periodically checking the fitness of each solution, so after generating the offspring ( 2 at a time) (crossover + mutation)
I need to check their validity. If any of the offspring is not valid I turn the set bits one by one into 0s starting from left to right( I tried selecting random indexes and checking
if they're set, after which I would turn them to 0, but for some reason it was taking too long to compile so I gave up). After validating the offspring I introduce them into the new
population, until the new population becomes as big as the old one. Due to the fact that I did not take the "keep invalid solutions" approach, I thought that keeping survivors was not
necessary, so no parents survive.

I have the table with the data below. One thing I've noticed when I was changing parameters was that the higher the generation count goes, the lower the average fitness is going to be,
but the maximum values tend to be either higher, or consistently higher.

note: input.txt is the file with 20 objects, input200.txt is the file with 200.
